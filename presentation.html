Slide 1 / Core AI Architecture

Agentic AI: Autonomous Workflows

Transforming unstructured enterprise chaos (Meeting Transcripts, HR Onboarding) into verifiable, managed operations.

01 / Ingestion

The system accepts unstructured human inputs through a high-performance Streamlit Portal, bridging the gap between chat and traditional ERP forms.

A. Meeting-to-Action Raw conversational transcripts containing implied deliverables.
B. HR Post-Onboarding Departmental selection requiring cross-referencing for buddy allocation.

02 / The Agent Swarm

🧠 Planner AgentDecomposes the prompt into a precise 6-step sequential operational pipeline.
📄 Understanding AgentExtracts nuanced tasks via Hybrid-AI (Gemini fallback to Native Rule-Engine).
👤 Assignment AgentIdentifies owners intelligently from database matrices + applies a confidence score.
✅ Validator AgentHard-gates execution; halts if ambiguity in task ownership requires human clarification.

03 / Execution

The execution tier isolates computational tasks from business logic, ensuring secure, trackable commits.

⚡ Executor Agent Pushes final tasks via securely credentialed APIs (Supabase DB, SMTP, Resend).
📋 Logger Agent Constructs a structured JSON payload of the entire multi-agent dialogue and archives it for absolute auditability.

Slide 2 / Hackathon USP

Enterprise Reliability & SLA Monitoring

Designed under the absolute assumption that APIs fail, dependencies timeout, and human tasks stall.

Asynchronous SLA Monitoring Agent

Runs asynchronously to system logic. Sweeps the persistent database to detect tasks marked "Unassigned" or stagnant beyond their allowed timeframe. Escalates "SLA BREACH ALERT 🚨" emails directly to IT Command chains automatically.

🔄 The Recovery Agent (Self-Healing)

Executors are wrapped in an exponential 3-strike retry loop. If a database transaction times out or SMTP server refuses connection, the Recovery Agent pauses, retries, and logs the degraded state intelligently without crashing the script.

Bulletproof Hybrid-AI Fallbacks

If the external Generative AI (Google Gemini / Ollama / Groq) runs out of quotas or RAM on the Render instance, the exception is intercepted natively, logging "Fallback Triggered", and parsing organically rolls over to a proprietary Regex Rule-Based Engine seamlessly.

100%
Traceable Agent Reasoning

Every AI decision (e.g. "Selected Employee B because Employee A was unavailable in Supabase") is logged immutably.

Stateless
Linux Node Deployment

Agents operate completely stateless in memory, capable of seamless horizontal scaling and multi-node Render deployments.
